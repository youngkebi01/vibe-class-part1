## 1. 제품 개요

본 문서는 로또 번호 추천 시스템의 신규 기능에 대한 요구사항을 정의합니다. 기존 바닐라 JavaScript 기반 구현(`lotto.html`)을 베이스로 하며, 사용자 맞춤형 번호 생성 경험을 제공하는 것이 핵심 목표입니다.

### 🎯 핵심 가치 제안

- 사용자가 직접 원하는 번호를 **포함하거나 제외**하면서 나만의 로또 번호를 생성
- **수동(직접 선택)** 과 **자동(시스템 생성)** 을 자유롭게 조합하는 하이브리드 방식 지원
- **추천 번호 클릭 한 번**으로 즉시 6개 번호 확정 — 빠르고 직관적인 UX 제공

## 2. 사용자 페르소나 및 문제 정의

### 2.1 페르소나

| 구분 | 페르소나 A — 신중한 선택자 | 페르소나 B — 빠른 결정자 |
|------|--------------------------|------------------------|
| **특징** | 특정 번호에 집착, 제외할 번호 명확 | 빨리 생성하고 싶음, 클릭 최소화 원함 |
| **니즈** | 내 번호는 포함, 싫은 번호는 제외 | 추천 번호 클릭 즉시 6개 확정 |
| **Pain Point** | 기존 완전랜덤은 원하지 않는 번호 포함 | 여러 단계 설정이 번거로움 |

### 2.2 해결하고자 하는 문제

- 기존 완전랜덤 모드는 사용자가 기피하는 번호(예: 13, 4)가 포함되어 불만족
- 행운 번호 포함 기능만으로는 '제외' 개념이 없어 세밀한 커스터마이징 불가
- 추천 번호를 보고 마음에 들면 바로 확정하는 **원클릭 플로우** 부재
- 수동으로 6개를 직접 고르는 옵션(완전 수동 모드)이 없음

## 3. 기능 요구사항

---

### `FEATURE 01`
### 3.1 제외 번호 선택 기능

사용자가 생성 결과에서 절대 나오지 않아야 할 번호를 사전에 지정하는 기능입니다.

#### 요구사항 상세

| ID | 요구사항 |
|----|---------|
| FR-01-1 | 1~45 중 최대 39개(6개는 반드시 남겨야 함)까지 제외 번호 지정 가능 |
| FR-01-2 | 피커 UI: 5행 9열 그리드(번호 1~45), 선택 시 빨간 취소선 + 반투명 처리 |
| FR-01-3 | 제외된 번호는 생성 알고리즘의 풀(pool)에서 완전히 제거 |
| FR-01-4 | 제외 번호와 행운 번호 충돌 시 경고 토스트 메시지 표시 및 행운 번호 우선 처리 |
| FR-01-5 | 제외 번호는 세션 내 유지, 초기화 버튼으로 일괄 해제 가능 |
| FR-01-6 | 현재 선택된 제외 번호 수 실시간 뱃지 표시 (예: `✕ 5개 제외중`) |

#### UI 동작 명세

- **제외 번호 버튼** 클릭 → 전체화면 오버레이 피커 오픈
- 번호 클릭 → 토글(선택/해제), 선택된 번호는 **빨간 배경 + 취소선**
- 최소 6개 번호는 항상 선택 가능 상태 유지 (제외 한계 도달 시 미선택 번호 비활성화)
- **확인** 버튼 → 저장 후 피커 닫기 / **취소** → 변경 무시

### `FEATURE 02`
### 3.2 추천 번호 선택 기능

시스템이 사전 계산된 후보 번호 세트를 제시하고, 사용자가 마음에 드는 세트를 클릭하여 즉시 확정하는 기능입니다.

#### 요구사항 상세

| ID | 요구사항 |
|----|---------|
| FR-02-1 | 추천 번호 버튼 클릭 시 3~5개의 후보 세트(6개 번호) 카드 형태로 표시 |
| FR-02-2 | 각 후보 세트는 레이블 포함 (예: '고빈도 조합', '균형 조합', '저빈도 조합') |
| FR-02-3 | 세트 카드 클릭 → 해당 6개 번호가 즉시 결과 영역에 확정 표시 |
| FR-02-4 | 추천 세트 생성 시 현재 설정된 제외 번호 필터링 반영 |
| FR-02-5 | '다른 추천 보기' 버튼으로 새로운 후보 세트 재생성 가능 |
| FR-02-6 | 확정된 번호는 복사, 저장 등 기존 결과 영역 기능 그대로 활용 가능 |

#### 추천 알고리즘 전략

| 세트명 | 알고리즘 설명 | 특징 |
|--------|-------------|------|
| 고빈도 조합 | 출현 가중치 상위 15개 번호 중 랜덤 6개 추출 | 통계 기반, 안정적 |
| 균형 조합 | 1~10 / 11~20 / 21~30 / 31~40 / 41~45 구간에서 균등 분산 | 번호 편중 없음 |
| 저빈도 조합 | 가중치 하위 15개 번호 중 랜덤 6개 추출 | 역발상, 비선호 번호 |
| 홀짝 균형 | 홀수 3개 + 짝수 3개 고정 비율로 추출 | 패턴 기반 |
| 랜덤 추천 | 순수 랜덤 생성 (완전 무작위) | 빠른 결정용 |

### `FEATURE 03`
### 3.3 수동 / 자동 하이브리드 번호 생성

사용자가 원하는 번호는 직접 선택(수동)하고 나머지는 시스템이 자동 채우는 하이브리드 방식을 지원합니다.

#### 요구사항 상세

| ID | 요구사항 |
|----|---------|
| FR-03-1 | 모드 탭에 '수동' 옵션 추가 — 기존 완전랜덤 / 행운포함 / 통계기반과 병렬 구성 |
| FR-03-2 | 수동 모드: 1~45 피커에서 사용자가 직접 0~6개 번호 선택 |
| FR-03-3 | 선택 번호가 6개 미만일 때 '자동 채우기' 버튼 활성화 → 나머지를 랜덤으로 자동 보완 |
| FR-03-4 | 선택 번호가 정확히 6개이면 '자동 채우기' 없이 즉시 확정 버튼 활성화 |
| FR-03-5 | 수동 선택 번호는 제외 번호와 중복 불가 (피커에서 제외 번호 선택 시 경고) |
| FR-03-6 | 자동 채우기 시 제외 번호 필터 자동 적용 |
| FR-03-7 | 수동 선택 영역 UI: 최대 6개 슬롯, 비어있는 슬롯은 '?' 표시, 클릭 시 피커 오픈 |

#### 수동 모드 상태 흐름

```
1. 수동 모드 탭 선택
2. 번호 슬롯(6개) 표시 — 초기에는 모두 비어있음(?)
3. 슬롯 클릭 → 피커 오픈 → 번호 선택 → 슬롯에 채워짐
4. 6개 미만 선택 상태에서 '번호 생성' 클릭 → 나머지 자동 채움
5. 6개 모두 수동 선택 시 → '번호 생성' 클릭 → 즉시 결과 확정
```

## 4. 기존 기능과의 연동

신규 기능은 기존 기능(완전랜덤, 행운번호 포함, 통계기반, 게임 수 선택, 복사)과 완전히 호환되어야 합니다.

| 기존 기능 | 연동 방식 | 주의사항 |
|---------|---------|---------|
| 완전랜덤 모드 | 제외 번호 필터 자동 반영 | 제외 번호로 풀이 작아져도 6개 생성 보장 |
| 행운번호 포함 모드 | 제외 번호와 충돌 방지 로직 추가 | 행운번호가 제외번호에 포함되면 경고 |
| 통계기반 모드 | 제외 번호 필터 + 가중치 재계산 | 제외된 번호의 가중치 0으로 처리 |
| 게임 수 (1/3/5) | 모든 신규 모드에서 동일하게 적용 | 추천번호 확정은 1게임 고정 |
| 번호 복사 | 신규 기능 결과도 동일 포맷으로 복사 | 수동/추천 모드 레이블 포함 |

## 5. UI/UX 명세

### 5.1 전체 레이아웃 구조

```
┌─────────────────────────────────────────┐
│  헤더: 타이틀 + 부제                       │
├─────────────────────────────────────────┤
│  모드 탭                                  │
│  [ 완전랜덤 | 행운번호 | 통계기반 | 수동 | 추천번호 ]  │
│                              ↑ 신규 2개   │
├─────────────────────────────────────────┤
│  게임 수: [ 1 | 3 | 5 ]                  │
├─────────────────────────────────────────┤
│  제외 번호 설정 바  ← 신규               │
│  [ 제외 번호 설정 버튼 ]  ✕13  ✕4  ✕7   │
├─────────────────────────────────────────┤
│  모드별 영역 (조건부 표시)                 │
│  · 행운번호 슬롯 (행운포함 모드)           │
│  · 수동 슬롯 6개 (수동 모드)  ← 신규     │
│  · 추천 후보 카드 (추천번호 모드)  ← 신규 │
├─────────────────────────────────────────┤
│  [ 번호 생성하기 버튼 ]                   │
├─────────────────────────────────────────┤
│  결과 영역: 티켓 카드 목록                 │
├─────────────────────────────────────────┤
│  [ 번호 복사 버튼 ]                       │
└─────────────────────────────────────────┘
```

### 5.2 제외 번호 UI 스펙

- **위치**: 모드 탭 아래, 게임 수 선택 위
- **기본 상태**: `제외 번호 설정` 버튼 + `없음` 텍스트
- **지정 후 상태**: 번호 뱃지 나열 (예: `✕13` `✕4` `✕7`), 각 뱃지 클릭 시 즉시 해제
- **최대 표시**: 뱃지 5개까지 인라인, 초과 시 `+N개 더` 표시

### 5.3 추천 번호 모드 UI 스펙

- 추천 후보 카드: 가로 스크롤 또는 세로 리스트 (모바일 대응)
- 각 카드: **세트명 레이블** + 6개 번호 볼 + `이 번호로 결정` 버튼
- 선택된 카드: 골드 테두리 + 체크 표시
- `다른 추천 보기` 버튼: 카드 목록 하단, 클릭 시 애니메이션과 함께 새 후보 생성

### 5.4 수동 모드 UI 스펙

- **6개 슬롯**: 각 슬롯은 번호 볼 크기(48px)로 시각적 일관성 유지
- **빈 슬롯**: 점선 원 + `?` 텍스트, 클릭 시 피커 오픈
- **채워진 슬롯**: 해당 번호 볼 표시, 클릭 시 슬롯 초기화
- **하단 상태**: `3/6 선택됨` 텍스트 + `자동으로 나머지 채우기` 버튼

## 7. 엣지 케이스 & 예외 처리

| 시나리오 | 처리 방식 |
|---------|---------|
| 제외 번호 40개 이상 설정 시도 | 39개 초과 불가 처리 — 피커에서 추가 선택 비활성화 |
| 제외 번호 + 행운번호 동일 번호 지정 | 경고 토스트: `'행운 번호는 제외될 수 없어요'` — 행운번호 우선 |
| 수동 모드에서 제외 번호 선택 시도 | 피커에서 해당 번호 회색 처리 + 클릭 시 안내 메시지 |
| 추천 번호 후보가 제외 번호와 충돌 | 생성 단계에서 자동 필터링, 사용자 개입 불필요 |
| 제외 번호 풀로 인해 6개 생성 불가 | 불가 상태 감지 후 `'제외 번호를 줄여주세요'` 오류 메시지 |
| 수동 6개 선택 + 게임 수 3 설정 | 3게임 모두 동일 6개 번호로 결과 표시 (중복 허용) |

## 8. 구현 가이드 (바닐라 JS)

### 8.1 상태(State) 구조

```javascript
const STATE = {
  mode: 'random',       // 'random' | 'lucky' | 'stats' | 'manual' | 'recommend'
  count: 1,             // 게임 수: 1 | 3 | 5
  luckyNums: [],        // 행운 번호 배열 (최대 6)
  excludeNums: [],      // 제외 번호 배열 (최대 39)
  manualSlots: [null, null, null, null, null, null],  // 수동 슬롯 6개
  recommendSets: [],    // 추천 후보 세트 목록
  selectedRecommend: null, // 선택된 추천 세트 인덱스
  generated: [],        // 생성된 결과
};
```

### 8.5 알고리즘: 충돌 감지

```javascript
function validateConflict() {
  const conflicts = STATE.luckyNums.filter(n => STATE.excludeNums.includes(n));
  if (conflicts.length > 0) {
    showToast(`⚠️ ${conflicts.join(', ')}번은 행운 번호라 제외할 수 없어요`);
    // 제외 번호에서 충돌 번호 제거 (행운 번호 우선)
    STATE.excludeNums = STATE.excludeNums.filter(n => !conflicts.includes(n));
  }
}
```
